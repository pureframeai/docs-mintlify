This page covers two related operations for managing webhook event deliveries: retrieving the paginated delivery history for a given endpoint, and retrying a previously failed delivery. Use delivery history to diagnose integration issues, inspect HTTP response codes from your server, and identify events that need to be replayed.

---

## List Deliveries

Retrieve a paginated list of all delivery attempts for a specific webhook endpoint, ordered by most recent first.

### Endpoint

```
GET https://api.pureframe.ai/v1/webhooks/{endpoint_id}/deliveries
```

**Authentication required:** Yes

### Path Parameters

<ParamField path="endpoint_id" type="string" required>
  The unique identifier of the webhook endpoint (e.g. `wh_abc123`).
</ParamField>

### Query Parameters

<ParamField query="page" type="integer">
  The page number to retrieve. Defaults to `1`.
</ParamField>

<ParamField query="per_page" type="integer">
  Number of delivery records per page. Defaults to `20`. Maximum is `50`.
</ParamField>

### Delivery Object Fields

<ResponseField name="id" type="string">
  Unique identifier for this delivery attempt (e.g. `del_xyz789`).
</ResponseField>

<ResponseField name="status" type="string">
  Delivery outcome. Either `"delivered"` (your server responded with a `2xx` status) or `"failed"` (your server returned a non-`2xx` status, timed out, or was unreachable).
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the delivery was attempted.
</ResponseField>

<ResponseField name="response_status_code" type="integer">
  The HTTP status code returned by your server during this delivery attempt. `null` if the request could not be completed (e.g. network timeout).
</ResponseField>

### Request Example

```bash
curl "https://api.pureframe.ai/v1/webhooks/wh_abc123/deliveries?page=1&per_page=20" \
  -H "Authorization: Bearer pf_your_key"
```

### Response Example

Returns `200 OK` with a paginated list of delivery objects.

```json
{
  "data": [
    {
      "id": "del_xyz789",
      "status": "delivered",
      "created_at": "2024-11-05T14:22:10Z",
      "response_status_code": 200
    },
    {
      "id": "del_uvw456",
      "status": "failed",
      "created_at": "2024-11-05T13:58:44Z",
      "response_status_code": 503
    }
  ],
  "page": 1,
  "per_page": 20,
  "total": 2
}
```

## Retry a Delivery

Re-attempt delivery of a specific failed event to the endpoint's current URL. This is useful for replaying events after you've resolved a server outage or fixed a processing bug.

### Endpoint

```
POST https://api.pureframe.ai/v1/webhooks/{endpoint_id}/deliveries/{delivery_id}/retry
```

**Authentication required:** Yes

### Path Parameters

<ParamField path="endpoint_id" type="string" required>
  The unique identifier of the webhook endpoint (e.g. `wh_abc123`).
</ParamField>

<ParamField path="delivery_id" type="string" required>
  The unique identifier of the delivery attempt to retry (e.g. `del_uvw456`).
</ParamField>

### Request Example

```bash
curl -X POST https://api.pureframe.ai/v1/webhooks/wh_abc123/deliveries/del_uvw456/retry \
  -H "Authorization: Bearer pf_your_key"
```

### Response

Returns `202 Accepted`. The retry is queued for asynchronous delivery.

```
HTTP/1.1 202 Accepted
```
