Register a new webhook endpoint so PureFrame can notify your application when specific events occur. Once created, PureFrame will begin delivering signed event payloads to the specified URL whenever a matching event is triggered — for example, when video processing completes and frames are ready to search.

## Endpoint

```
POST https://api.pureframe.ai/v1/webhooks
```

**Authentication required:** Yes  
**Content-Type:** `application/json`

## Request Parameters

<ParamField body="url" type="string" required>
  The HTTPS URL that will receive webhook event payloads. Must use `https://`. HTTP endpoints are not accepted.
</ParamField>

<ParamField body="events" type="array of strings" required>
  One or more event type strings to subscribe to. For example: `["video.processed"]`. PureFrame will only deliver events matching the types listed here.
</ParamField>

## Request Example

```bash
curl -X POST https://api.pureframe.ai/v1/webhooks \
  -H "Authorization: Bearer pf_your_key" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://yourapp.com/hooks/pureframe", "events": ["video.processed"]}'
```

## Response

Returns `201 Created` with the newly created webhook endpoint object.

<ResponseField name="id" type="string">
  Unique identifier for the newly created endpoint (e.g. `wh_abc123`).
</ResponseField>

<ResponseField name="url" type="string">
  The HTTPS destination URL you registered.
</ResponseField>

<ResponseField name="events" type="array of strings">
  The event types this endpoint is subscribed to.
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Indicates whether the endpoint is active. Newly created endpoints are `true` by default.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the endpoint was created.
</ResponseField>

## Response Example

```json
{
  "id": "wh_abc123",
  "url": "https://yourapp.com/hooks/pureframe",
  "events": ["video.processed"],
  "is_active": true,
  "created_at": "2024-11-01T10:00:00Z"
}
```
