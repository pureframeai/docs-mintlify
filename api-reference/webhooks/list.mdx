Retrieve all webhook endpoints registered to your PureFrame account. The response includes both active and inactive endpoints so you can audit your current webhook configuration, identify stale subscriptions, and manage delivery targets across your integration.

## Endpoint

<ParamField header="Authorization" type="string" required>
  Bearer token using your PureFrame API key (`pf_...`).
</ParamField>

```
GET https://api.pureframe.ai/v1/webhooks
```

**Authentication required:** Yes

**Request parameters:** None

## Request Example

```bash
curl https://api.pureframe.ai/v1/webhooks \
  -H "Authorization: Bearer pf_your_key"
```

## Response

Returns an array of webhook endpoint objects.

<ResponseField name="id" type="string">
  Unique identifier for the webhook endpoint (e.g. `wh_abc123`).
</ResponseField>

<ResponseField name="url" type="string">
  The HTTPS destination URL that receives event payloads.
</ResponseField>

<ResponseField name="events" type="array of strings">
  The list of event types this endpoint is subscribed to (e.g. `["video.processed"]`).
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Whether the endpoint is currently active. Inactive endpoints do not receive deliveries.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp indicating when the endpoint was created.
</ResponseField>

## Response Example

```json
[
  {
    "id": "wh_abc123",
    "url": "https://yourapp.com/hooks/pureframe",
    "events": ["video.processed"],
    "is_active": true,
    "created_at": "2024-11-01T10:00:00Z"
  },
  {
    "id": "wh_def456",
    "url": "https://yourapp.com/hooks/pureframe-staging",
    "events": ["video.processed"],
    "is_active": false,
    "created_at": "2024-09-15T08:30:00Z"
  }
]
```
