Retrieve a paginated list of videos belonging to a collection. You can filter results by processing status to, for example, show only videos that have finished indexing or isolate any that have failed. Each item in the response includes metadata such as dimensions, duration, codec, and file size so you can build rich media library UIs without additional requests.

## Endpoint

**GET** `https://api.pureframe.ai/v1/videos`

**Authentication:** Required — `Authorization: Bearer pf_...`

## Query Parameters

<ParamField query="collection_id" type="string" required>
  Return only videos belonging to this collection. You must provide a collection ID — cross-collection listing is not supported on this endpoint.
</ParamField>

<ParamField query="status" type="string">
  Filter videos by processing status. Accepts a comma-separated list of one or more values: `queued`, `processing`, `done`, `failed`.

  **Example:** `status=done,failed`
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number to retrieve, starting at `1`.
</ParamField>

<ParamField query="per_page" type="integer" default="20">
  Number of videos to return per page. Minimum `1`, maximum `100`.
</ParamField>

## Request Example

```bash
curl -G https://api.pureframe.ai/v1/videos \
  -H "Authorization: Bearer pf_your_key" \
  --data-urlencode "collection_id=col_abc123" \
  --data-urlencode "status=done" \
  --data-urlencode "page=1" \
  --data-urlencode "per_page=20"
```

## Response — 200 OK

Returns a `data` array of `VideoListItem` objects and a `meta` object with pagination details.

<ResponseField name="data" type="array">
  Array of `VideoListItem` objects.

  <Expandable title="VideoListItem fields">
    <ResponseField name="id" type="string">
      Unique identifier for the video (e.g. `vid_7hWnBcYdZq`).
    </ResponseField>

    <ResponseField name="filename" type="string">
      Original filename as uploaded (e.g. `product-demo.mp4`).
    </ResponseField>

    <ResponseField name="collection_id" type="string">
      ID of the collection this video belongs to.
    </ResponseField>

    <ResponseField name="status" type="string">
      Current processing status: `queued`, `processing`, `done`, or `failed`.
    </ResponseField>

    <ResponseField name="thumbnail_url" type="string">
      URL of an auto-generated thumbnail image. Available once status is `done`.
    </ResponseField>

    <ResponseField name="duration_secs" type="number">
      Video duration in seconds.
    </ResponseField>

    <ResponseField name="width" type="integer">
      Video width in pixels.
    </ResponseField>

    <ResponseField name="height" type="integer">
      Video height in pixels.
    </ResponseField>

    <ResponseField name="fps" type="number">
      Frames per second of the video.
    </ResponseField>

    <ResponseField name="file_size_bytes" type="integer">
      Size of the original uploaded file in bytes.
    </ResponseField>

    <ResponseField name="codec" type="string">
      Video codec detected during processing (e.g. `h264`, `vp9`).
    </ResponseField>

    <ResponseField name="mime_type" type="string">
      MIME type of the uploaded file (e.g. `video/mp4`).
    </ResponseField>

    <ResponseField name="storage_bytes" type="integer">
      Total storage consumed by this video and its indexed data in bytes.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of when the video was uploaded.
    </ResponseField>

    <ResponseField name="progress_pct" type="integer">
      Indexing progress from `0` to `100`. Reaches `100` when status is `done`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Pagination and request metadata.

  <Expandable title="meta fields">
    <ResponseField name="page" type="integer">
      Current page number.
    </ResponseField>

    <ResponseField name="per_page" type="integer">
      Number of items per page.
    </ResponseField>

    <ResponseField name="total" type="integer">
      Total number of videos matching the query filters.
    </ResponseField>

    <ResponseField name="request_id" type="string">
      Unique identifier for this API request, useful for support.
    </ResponseField>
  </Expandable>
</ResponseField>
