Fetch the complete metadata record for a single video, including its current processing status, all technical properties, a presigned playback URL, and the total number of indexed segments available for search. This endpoint is the right place to check whether a video is ready and to obtain a fresh playback URL before rendering it in a player.

## Endpoint

**GET** `https://api.pureframe.ai/v1/videos/{video_id}`

**Authentication:** Required — `Authorization: Bearer pf_...`

## Path Parameters

<ParamField path="video_id" type="string" required>
  The unique identifier of the video to retrieve (e.g. `vid_7hWnBcYdZq`). Returned by `POST /v1/upload` and `GET /v1/videos`.
</ParamField>

## Request Example

```bash
curl https://api.pureframe.ai/v1/videos/vid_7hWnBcYdZq \
  -H "Authorization: Bearer pf_your_key"
```

## Response — 200 OK

Returns a `VideoDetail` object containing all metadata fields plus playback and segment information.

<ResponseField name="id" type="string">
  Unique identifier for the video.
</ResponseField>

<ResponseField name="filename" type="string">
  Original filename as uploaded.
</ResponseField>

<ResponseField name="collection_id" type="string">
  ID of the collection this video belongs to.
</ResponseField>

<ResponseField name="status" type="string">
  Current processing status: `queued`, `processing`, `done`, or `failed`.
</ResponseField>

<ResponseField name="thumbnail_url" type="string">
  URL of the auto-generated thumbnail image. Available once status is `done`.
</ResponseField>

<ResponseField name="duration_secs" type="number">
  Video duration in seconds.
</ResponseField>

<ResponseField name="width" type="integer">
  Video width in pixels.
</ResponseField>

<ResponseField name="height" type="integer">
  Video height in pixels.
</ResponseField>

<ResponseField name="fps" type="number">
  Frames per second of the source video.
</ResponseField>

<ResponseField name="file_size_bytes" type="integer">
  Size of the original uploaded file in bytes.
</ResponseField>

<ResponseField name="codec" type="string">
  Video codec detected during processing (e.g. `h264`, `vp9`).
</ResponseField>

<ResponseField name="mime_type" type="string">
  MIME type of the uploaded file (e.g. `video/mp4`).
</ResponseField>

<ResponseField name="storage_bytes" type="integer">
  Total storage consumed by this video and its indexed data in bytes.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the video was uploaded.
</ResponseField>

<ResponseField name="progress_pct" type="integer">
  Indexing progress from `0` to `100`. Reaches `100` when status is `done`.
</ResponseField>

<ResponseField name="playback_url" type="string">
  A presigned URL for streaming or downloading the video, valid for approximately **1 hour** from the time of the API call.
</ResponseField>

<ResponseField name="segments_count" type="integer">
  Total number of indexed segments available for this video. Only meaningful when status is `done`.
</ResponseField>

<Note>
  `playback_url` is a presigned URL valid for approximately 1 hour. Fetch a fresh URL from this endpoint before each playback session rather than caching it — stale URLs will result in a 403 response from the storage layer.
</Note>
