Upload a video file to a PureFrame collection and begin asynchronous indexing. Once submitted, PureFrame transcodes, segments, and embeds the video so it becomes searchable. Because indexing can take time depending on video length, the response returns immediately with a `job_id` — use it to poll for progress or supply a `webhook_url` to receive a push notification when processing completes.

## Endpoint

<ParamField path="method" initialValue="POST /v1/upload" />

**POST** `https://api.pureframe.ai/v1/upload`

**Content-Type:** `multipart/form-data`

**Authentication:** Required — `Authorization: Bearer pf_...`

## Request Parameters

<ParamField body="collection_id" type="string" required>
  The ID of the collection to add the uploaded video to. The collection must already exist. You can create collections via `POST /v1/collections`.
</ParamField>

<ParamField body="file" type="binary" required>
  The video file to upload. Accepted formats: **MP4, MOV, AVI, WebM, MPEG, MKV**. Maximum file size is determined by your plan quota.
</ParamField>

<ParamField body="webhook_url" type="string">
  An HTTPS URL to notify via a `POST` request when processing completes. The request body will contain the final job object including `status` and `video_id`. If omitted, poll `GET /v1/jobs/{job_id}` instead.
</ParamField>

## Request Example

```bash
curl -X POST https://api.pureframe.ai/v1/upload \
  -H "Authorization: Bearer pf_your_key" \
  -F "collection_id=col_abc123" \
  -F "file=@video.mp4"
```

## Response — 202 Accepted

PureFrame accepts the upload and queues it for indexing. The response body contains an `UploadResult` object.

<ResponseField name="job_id" type="string">
  Unique identifier for the background processing job. Pass this to `GET /v1/jobs/{job_id}` to poll for status and progress.
</ResponseField>

<ResponseField name="video_id" type="string">
  Unique identifier assigned to the video. Available immediately, even before indexing is complete.
</ResponseField>

<ResponseField name="status" type="string">
  Initial processing status. Always `"queued"` on a successful upload response.
</ResponseField>

```json
{
  "data": {
    "job_id": "job_9xkT2mLpQr",
    "video_id": "vid_7hWnBcYdZq",
    "status": "queued"
  },
  "meta": {
    "request_id": "req_4aFgHjKmNp"
  }
}
```

<Note>
  Poll `GET /v1/jobs/{job_id}` to track indexing progress. The `status` field follows this lifecycle:

  `queued` → `processing` → `done` (or `failed`)

  A `progress_pct` field (0–100) is available on the job object while status is `processing`.
</Note>
