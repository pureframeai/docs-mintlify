title: "Get Video Processing Job Status"

After uploading a video, PureFrame processes it asynchronously. Use this endpoint to poll the status of that background job and track indexing progress. The response includes the current lifecycle status, a percentage indicating how far along processing has advanced, and — in the case of a failure — a human-readable error message explaining what went wrong.

## Endpoint

**GET** `https://api.pureframe.ai/v1/jobs/{job_id}`

**Authentication:** Required — `Authorization: Bearer pf_...`

## Path Parameters

<ParamField path="job_id" type="string" required>
  The unique identifier of the processing job to retrieve. Returned as `job_id` in the `POST /v1/upload` response (e.g. `job_9xkT2mLpQr`).
</ParamField>

## Request Example

```bash
curl https://api.pureframe.ai/v1/jobs/job_9xkT2mLpQr \
  -H "Authorization: Bearer pf_your_key"
```

## Response — 200 OK

<ResponseField name="id" type="string">
  Unique identifier for this processing job.
</ResponseField>

<ResponseField name="video_id" type="string">
  The ID of the video being processed by this job.
</ResponseField>

<ResponseField name="user_id" type="string">
  The ID of the user account that initiated the upload.
</ResponseField>

<ResponseField name="status" type="string">
  Current lifecycle status of the job. One of:
  - `queued` — job is waiting to be picked up by a worker
  - `processing` — video is actively being transcoded and indexed
  - `done` — indexing completed successfully; the video is searchable
  - `failed` — an error occurred; see the `error` field for details
</ResponseField>

<ResponseField name="progress_pct" type="integer" default="0">
  Processing progress as an integer from `0` to `100`. Defaults to `0` when the job is first queued. Increments during the `processing` phase and reaches `100` when status transitions to `done`.
</ResponseField>

<ResponseField name="error" type="string | null">
  A human-readable description of what went wrong, or `null` if the job has not failed.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the job was created (i.e. when the video was uploaded).
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the most recent status update.
</ResponseField>

## Status Lifecycle

```
queued  →  processing  →  done
                       ↘  failed
```

Poll this endpoint periodically (e.g. every 5 seconds) until `status` is `done` or `failed`. Alternatively, supply a `webhook_url` during upload and PureFrame will POST the final job object to your server automatically.

## Python Polling Example

```python
import time
import requests

API_KEY = "pf_your_key"
JOB_ID  = "job_9xkT2mLpQr"

headers = {"Authorization": f"Bearer {API_KEY}"}

while True:
    response = requests.get(
        f"https://api.pureframe.ai/v1/jobs/{JOB_ID}",
        headers=headers,
    )
    response.raise_for_status()
    job = response.json()["data"]

    print(f"Status: {job['status']}  |  Progress: {job['progress_pct']}%")

    if job["status"] == "done":
        print(f"Video {job['video_id']} is ready for search!")
        break
    elif job["status"] == "failed":
        raise RuntimeError(f"Indexing failed: {job['error']}")

    time.sleep(5)
```
