Invoke a PureFrame tool directly by name from your AI agent or LLM orchestration layer. When your LLM decides to call a PureFrame tool — based on the schema returned by [`GET /v1/agent/schema.json`](/api-reference/agent/schema) — forward the tool name and input arguments to this endpoint. PureFrame executes the tool and returns structured JSON results, including base64-encoded JPEG thumbnails where applicable so vision-capable models can interpret visual content inline.

## Endpoint

```
POST https://api.pureframe.ai/v1/agent/call
```

**Authentication required:** Yes  
**Content-Type:** `application/json`

## Request Parameters

<ParamField body="tool" type="string" required>
  The name of the PureFrame tool to invoke. Must be one of: `search_videos`, `list_collections`, `get_collection`, `get_video`.
</ParamField>

<ParamField body="input" type="object">
  The input arguments for the tool, as an object. The required and optional keys vary by tool — refer to the [Agent Schema](/api-reference/agent/schema) for each tool's parameter definition.
</ParamField>

## Request Example

```bash
curl -X POST https://api.pureframe.ai/v1/agent/call \
  -H "Authorization: Bearer pf_your_key" \
  -H "Content-Type: application/json" \
  -d '{"tool": "search_videos", "input": {"query": "person waving", "collection_id": "col_abc123"}}'
```

## Response

Returns tool-specific JSON. The response structure depends on which tool was invoked.

For `search_videos`, results include a `thumbnail_base64` field on each match — a base64-encoded JPEG of the best matching frame. This can be passed directly as image input to vision-capable models.

<Note>
  `search_videos` results include `thumbnail_base64`, a base64-encoded JPEG that GPT-4o and Claude can consume directly as vision input. Pass it in an image content block alongside the tool result to let the model visually verify or describe the matched frame.
</Note>
