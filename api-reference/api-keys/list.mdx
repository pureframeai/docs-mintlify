Retrieve metadata for all API keys associated with your PureFrame account, including keys that have been revoked. This endpoint is useful for auditing access, verifying key names and permissions, and identifying when keys were last used. For security reasons, the full key value is never returned — only a short prefix is exposed to help you identify each key.

## Endpoint

```
GET https://api.pureframe.ai/v1/api_keys
```

**Authentication required:** Yes  
**Request parameters:** None

## Request Example

```bash
curl https://api.pureframe.ai/v1/api_keys \
  -H "Authorization: Bearer pf_your_key"
```

## Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the API key (e.g. `key_abc123`).
</ResponseField>

<ResponseField name="user_id" type="string">
  The identifier of the PureFrame account that owns this key.
</ResponseField>

<ResponseField name="name" type="string">
  The human-readable label assigned to the key when it was created.
</ResponseField>

<ResponseField name="prefix" type="string">
  The first several characters of the key (e.g. `pf_a1b2`). Use this to identify a key without exposing the full secret.
</ResponseField>

<ResponseField name="permissions" type="string">
  The permission scope of the key. Either `"all"` (full read/write access) or `"read_only"` (search and list operations only).
</ResponseField>

<ResponseField name="last_used_at" type="string | null">
  ISO 8601 timestamp of the most recent successful request made with this key. `null` if the key has never been used.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the key was created.
</ResponseField>

<ResponseField name="revoked_at" type="string | null">
  ISO 8601 timestamp of when the key was revoked. `null` if the key is still active.
</ResponseField>

<Note>
  The full key value is never returned after creation. Only the prefix is shown in listing responses. If you've lost a key, you'll need to revoke it and create a new one.
</Note>
