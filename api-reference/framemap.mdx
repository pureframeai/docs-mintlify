The FrameMap endpoint generates a visual index of representative frames across all indexed videos in a collection. Use it to give users a browsable mosaic of your video library without requiring a search query.

## Endpoint

```
GET /v1/framemap
```

**Authentication:** Bearer API key required (`Authorization: Bearer pf_...`)

## Query Parameters

<ParamField query="collection_id" type="string">
  Filter the FrameMap to a specific collection. If omitted, includes frames from all collections on your account.
</ParamField>

<ParamField query="force" type="boolean" default="false">
  Set to `true` to regenerate the FrameMap even if a cached version already exists. Useful after uploading new videos.
</ParamField>

## Request Example

```bash
curl "https://api.pureframe.ai/v1/framemap?collection_id=col_abc123" \
  -H "Authorization: Bearer pf_your_api_key"
```

Force a fresh rebuild:

```bash
curl "https://api.pureframe.ai/v1/framemap?collection_id=col_abc123&force=true" \
  -H "Authorization: Bearer pf_your_api_key"
```

## Response

Returns a JSON object describing the visual frame index for the requested collection. The exact structure depends on the number of indexed videos.

```json
{
  "frames": [
    {
      "video_id": "vid_ghi789",
      "timestamp": 4.2,
      "thumbnail_url": "https://storage.pureframe.ai/thumbnails/vid_ghi789_4s.jpg"
    },
    {
      "video_id": "vid_ghi789",
      "timestamp": 18.0,
      "thumbnail_url": "https://storage.pureframe.ai/thumbnails/vid_ghi789_18s.jpg"
    }
  ]
}
```

<Tip>
  Call FrameMap with `force=true` after uploading and indexing new videos to ensure the latest frames are included in the index.
</Tip>
