Retrieve a paginated list of all collections belonging to your account, sorted by creation date in descending order (newest first). Each item in the response includes a summary of the collection's contents, including the total number of videos and a set of representative thumbnail URLs suitable for building gallery-style interfaces.

## Endpoint

```http
GET /v1/collections
```

**Authentication required:** Yes — `Authorization: Bearer pf_your_key`

## Query Parameters

This endpoint accepts the standard pagination parameters. No additional filtering parameters are supported.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `page` | integer | `1` | Page number to retrieve. |
| `per_page` | integer | `20` | Number of collections per page. Maximum `100`. |

## Response

Returns an array of `CollectionListItem` objects inside the standard `data` envelope.

### CollectionListItem Fields

<ResponseField name="id" type="string">
  Unique identifier for the collection (e.g. `col_01hx9z2b3m4n5p6q7r8s9t0u`).
</ResponseField>

<ResponseField name="name" type="string">
  The human-readable name of the collection. Unique per account.
</ResponseField>

<ResponseField name="video_count" type="integer">
  Total number of videos currently stored in this collection, including those still being indexed.
</ResponseField>

<ResponseField name="thumbnail_urls" type="array of strings">
  Up to four representative thumbnail image URLs sampled from videos in the collection. Empty array if the collection contains no videos.
</ResponseField>

<ResponseField name="created_at" type="datetime">
  ISO 8601 timestamp indicating when the collection was created (e.g. `2024-03-15T10:30:00Z`).
</ResponseField>
