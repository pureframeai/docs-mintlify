Create a new collection to group and organize your videos. Collections act as top-level containers — every video you upload or index belongs to exactly one collection. Once created, a collection can be renamed, and videos can be moved between collections. You can create as many collections as your plan allows.

## Endpoint

```http
POST /v1/collections
```

**Authentication required:** Yes — `Authorization: Bearer pf_your_key`  
**Content-Type:** `application/json`

## Request Body

<ParamField body="name" type="string" required>
  A unique, human-readable name for the collection. Must be between 1 and 255 characters. Names are unique per account — attempting to create a collection with a name that already exists will return a `422` error.
</ParamField>

## Request Example

<CodeGroup>

```bash cURL
curl --request POST \
  --url "https://api.pureframe.ai/v1/collections" \
  --header "Authorization: Bearer pf_your_key" \
  --header "Content-Type: application/json" \
  --data '{
    "name": "Product Demos"
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.pureframe.ai/v1/collections",
    headers={
        "Authorization": "Bearer pf_your_key",
        "Content-Type": "application/json",
    },
    json={"name": "Product Demos"},
)
print(response.json())
```

## Response

Returns the newly created `Collection` object with HTTP `201 Created`.

<ResponseField name="id" type="string">
  Unique identifier for the newly created collection (e.g. `col_01hx9z2b3m4n5p6q7r8s9t0u`).
</ResponseField>

<ResponseField name="user_id" type="string">
  The ID of the account that owns this collection.
</ResponseField>

<ResponseField name="name" type="string">
  The name provided when creating the collection.
</ResponseField>

<ResponseField name="created_at" type="datetime">
  ISO 8601 timestamp of when the collection was created.
</ResponseField>

<Note>
  If the provided `name` conflicts with an existing collection on your account, the API returns `422 Validation Error` with a `detail` entry pointing to the `name` field. Choose a different name or retrieve the existing collection's ID using [List Collections](/api-reference/collections/list).
</Note>
