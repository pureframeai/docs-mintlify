title: "Search Video Segments by Text or Image"

Query your indexed video collections using natural language text, a reference image file, or an image URL. PureFrame returns a ranked list of timestamped video segments whose visual content, scene context, or transcript semantically matches your input. Results include presigned clip URLs so you can stream matching moments directly — no additional requests needed. Combine multiple search modes to surface the richest, most relevant matches across different layers of your videos.

## Endpoint

**POST** `https://api.pureframe.ai/v1/search`

**Content-Type:** `multipart/form-data`

**Authentication:** Required — `Authorization: Bearer pf_...`

<Note>
  At least one of `query`, `image`, or `image_url` is required. You may combine a text `query` with an `image` or `image_url` for multimodal search.
</Note>

## Request Parameters

<ParamField body="query" type="string">
  A text string describing what to search for. Can be a keyword, phrase, or — when `natural_language` is `true` — a full natural-language sentence such as *"a presenter writing on a whiteboard"*.
</ParamField>

<ParamField body="image" type="file">
  A reference image file for visual similarity search. PureFrame will return video segments that visually resemble the provided image. Accepted formats: JPEG, PNG, WebP, GIF.
</ParamField>

<ParamField body="image_url" type="string">
  A publicly accessible URL pointing to a reference image. Used as an alternative to uploading an `image` file directly.
</ParamField>

<ParamField body="collection_id" type="string">
  Scope the search to a single collection. If omitted, PureFrame searches across all collections accessible to your API key.
</ParamField>

<ParamField body="modes" type="array" default='["video","transcript","scene"]'>
  An array of search modes to run. Available values:
  - `video` — match against visual frame embeddings
  - `transcript` — match against spoken-word transcription text
  - `scene` — match against scene-level semantic descriptions

  Providing a subset focuses the search and can improve precision for specific use cases.
</ParamField>

<ParamField body="natural_language" type="boolean" default="false">
  When `true`, PureFrame treats the `query` as a full natural-language sentence and uses a more expressive semantic model optimised for descriptive queries rather than keyword lookups.
</ParamField>

<ParamField body="labels" type="array">
  Filter results to segments that have been tagged with all of the specified semantic labels. Labels are strings such as `"outdoor"`, `"person"`, or `"product-demo"`.
</ParamField>

<ParamField body="page" type="integer" default="1">
  Page number for paginating results, starting at `1`.
</ParamField>

<ParamField body="per_page" type="integer" default="10">
  Number of result videos to return per page. Each result may contain multiple matching segments. Maximum `50`.
</ParamField>

## Request Example

```bash
curl -X POST https://api.pureframe.ai/v1/search \
  -H "Authorization: Bearer pf_your_key" \
  -F "query=person writing on whiteboard" \
  -F "collection_id=col_abc123" \
  -F "modes[]=video" \
  -F "modes[]=transcript" \
  -F "natural_language=false" \
  -F "per_page=10"
```

## Response — 200 OK

Returns a `data` array where each element is a `SearchResult` grouping a matched video with its relevant segments.

<ResponseField name="data" type="array">
  Array of `SearchResult` objects, one per matched video, ordered by the score of the top-matching segment.

  <Expandable title="SearchResult fields">
    <ResponseField name="video_id" type="string">
      Unique identifier of the matched video.
    </ResponseField>

    <ResponseField name="filename" type="string">
      Original filename of the matched video.
    </ResponseField>

    <ResponseField name="collection_id" type="string">
      ID of the collection this video belongs to.
    </ResponseField>

    <ResponseField name="status" type="string">
      Processing status of the video. Will always be `done` for videos returned in search results.
    </ResponseField>

    <ResponseField name="segments" type="array">
      Ordered list of matching segments within this video, sorted by descending score.

      <Expandable title="Segment fields">
        <ResponseField name="timestamp_start" type="number">
          Start time of the segment in seconds from the beginning of the video.
        </ResponseField>

        <ResponseField name="timestamp_end" type="number">
          End time of the segment in seconds from the beginning of the video.
        </ResponseField>

        <ResponseField name="score" type="number">
          Similarity score between `0.0` and `1.0`. Higher values indicate stronger relevance to the query.
        </ResponseField>

        <ResponseField name="type" type="string">
          The search mode that produced this match: `video`, `transcript`, or `scene`.
        </ResponseField>

        <ResponseField name="text_content" type="string | null">
          For `transcript`-type segments, the verbatim spoken text within the segment window. `null` for `video` and `scene` segments.
        </ResponseField>

        <ResponseField name="video_url" type="string">
          A presigned URL to stream the video clip for this segment, valid for approximately **1 hour**.
        </ResponseField>

        <ResponseField name="thumbnail_url" type="string">
          URL of a thumbnail image representing the segment's key frame.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Pagination and request metadata.

  <Expandable title="meta fields">
    <ResponseField name="page" type="integer">Current page number.</ResponseField>
    <ResponseField name="per_page" type="integer">Items per page.</ResponseField>
    <ResponseField name="total" type="integer">Total number of matching videos.</ResponseField>
    <ResponseField name="request_id" type="string">Unique identifier for this request.</ResponseField>
  </Expandable>
</ResponseField>

<Tip>
  Scores close to `1.0` indicate strong matches. Use `score` as a threshold to filter low-confidence results — for most use cases, a threshold of `0.7` or higher surfaces reliably relevant segments while excluding weak associations.
</Tip>
