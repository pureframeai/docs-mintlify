---

Signal to PureFrame whether a particular search result was relevant or not. Feedback is used to fine-tune result ranking specifically for your account, so over time search results reflect the preferences and domain vocabulary of your users. Submitting both positive signals (results that were clicked or used) and negative signals (results that were dismissed or unhelpful) produces the most effective improvements.

## Endpoint

**POST** `https://api.pureframe.ai/v1/search/feedback`

**Content-Type:** `application/x-www-form-urlencoded`

**Authentication:** Required — `Authorization: Bearer pf_...`

## Request Parameters

<ParamField body="video_id" type="string" required>
  The ID of the video containing the segment you are providing feedback on. Matches the `video_id` field in the search response.
</ParamField>

<ParamField body="timestamp_start" type="number" required>
  The start timestamp (in seconds) of the specific segment you are rating. Matches the `timestamp_start` value from the search result segment.
</ParamField>

<ParamField body="signal" type="string" required>
  The feedback signal. Must be one of:
  - `"positive"` — the result was relevant and useful
  - `"negative"` — the result was irrelevant or misleading
</ParamField>

<ParamField body="query" type="string">
  The original search query that produced this result. Including the query allows PureFrame to correlate the feedback with the specific search context and improves tuning accuracy.
</ParamField>

## Request Example

```bash
curl -X POST https://api.pureframe.ai/v1/search/feedback \
  -H "Authorization: Bearer pf_your_key" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "video_id=vid_7hWnBcYdZq" \
  --data-urlencode "timestamp_start=34.2" \
  --data-urlencode "signal=positive" \
  --data-urlencode "query=person writing on whiteboard"
```

## Response — 200 OK

A successful submission returns `200 OK`. The `data` field is `null` — there is no feedback object to return. The `meta` object contains the request ID for tracing.

```json
{
  "data": null,
  "meta": {
    "request_id": "req_7dEjFlGnHo"
  }
}
```
