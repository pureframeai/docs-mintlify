The PureFrame REST API gives you programmatic access to video ingestion, semantic search, collection management, and webhook notifications. Every feature available in the PureFrame dashboard is also available through this API, making it straightforward to integrate video intelligence directly into your applications.

## Base URL

All API requests are made to the following base URL. Every endpoint is versioned under `/v1/`.

```
https://api.pureframe.ai/v1
```

## Authentication

Every request must include your API key as a Bearer token in the `Authorization` header. You can generate and manage API keys from your [PureFrame dashboard](/authentication).

```http
Authorization: Bearer pf_your_key
```

<Note>
  Keep your API key secret. Do not expose it in client-side code or public repositories. If a key is compromised, revoke it immediately from your dashboard and issue a new one.
</Note>

## Response Envelope

Every response from the PureFrame API is wrapped in a consistent envelope with two top-level keys:

| Key | Description |
|-----|-------------|
| `data` | The primary payload â€” an object or array depending on the endpoint. |
| `meta` | Request metadata including a unique request ID, server-side processing time, and pagination details. |

```json
{
  "data": { "..." },
  "meta": {
    "request_id": "req_abc123",
    "took_ms": 42,
    "total": null,
    "page": null,
    "per_page": null
  }
}
```

### Meta Fields

| Field | Type | Description |
|-------|------|-------------|
| `request_id` | string | Unique identifier for this request. Include this in support requests. |
| `took_ms` | integer | Server-side processing time in milliseconds. |
| `total` | integer \| null | Total number of items available. Populated on list endpoints. |
| `page` | integer \| null | Current page number. Populated on list endpoints. |
| `per_page` | integer \| null | Number of items per page. Populated on list endpoints. |
| `facets` | object \| null | Aggregated facet counts. Populated on search endpoints. |

## Error Format

When a request fails, PureFrame returns an appropriate HTTP `4xx` or `5xx` status code alongside a JSON body describing what went wrong.

```json
{
  "error": {
    "code": "validation_error",
    "message": "Request body validation failed.",
    "detail": [
      {
        "field": "name",
        "issue": "Field is required and cannot be blank."
      }
    ]
  }
}
```

## HTTP Status Codes

| Status | Meaning |
|--------|---------|
| `200 OK` | The request succeeded and the response body contains the result. |
| `201 Created` | A new resource was successfully created. |
| `202 Accepted` | The request was accepted for asynchronous processing (e.g. video indexing). |
| `204 No Content` | The request succeeded and there is no response body (e.g. delete operations). |
| `401 Unauthorized` | The `Authorization` header is missing or the API key is invalid. |
| `403 Forbidden` | The API key is valid but does not have permission to perform this action. |
| `404 Not Found` | The requested resource does not exist. |
| `422 Validation Error` | The request body failed validation. The `detail` array describes each field-level issue. |
| `429 Too Many Requests` | You have exceeded your rate limit. Check the `Retry-After` header for when to resume. |

## Pagination

List endpoints return paginated results. Control pagination with the following query parameters:

| Parameter | Default | Maximum | Description |
|-----------|---------|---------|-------------|
| `page` | `1` | â€” | The page number to retrieve. |
| `per_page` | `20` | `100` | Number of items to return per page. |

Pagination details are reflected in the `meta` object of every list response (`total`, `page`, `per_page`).

```http
GET /v1/collections?page=2&per_page=50
```

## API Versioning

All endpoints are prefixed with `/v1/`. When breaking changes are introduced, a new version prefix will be released and the previous version will continue to work during a documented deprecation window.
