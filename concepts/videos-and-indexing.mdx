When you upload a video to PureFrame, it goes through an asynchronous indexing pipeline. PureFrame extracts visual frames, runs speech-to-text transcription, and detects scene boundaries — then stores searchable embeddings for each. Once indexing finishes, every frame, spoken word, and scene in the video becomes instantly searchable.

## Asynchronous processing

Video indexing takes time, so the upload endpoint returns immediately with a job `id` rather than waiting for processing to complete. Use that `id` to poll the jobs endpoint and track progress.

```bash
# 1. Upload a video
curl -X POST https://api.pureframe.ai/v1/videos \
  -H "Authorization: Bearer pf_..." \
  -F "file=@/path/to/video.mp4" \
  -F "collection_id=col_01hx3k9z2f8r4e7w"
```

```json
{
  "id": "job_01hx4m2p7n9q6t3r",
  "status": "queued"
}
```

```bash
# 2. Poll for status
curl https://api.pureframe.ai/v1/jobs/job_01hx4m2p7n9q6t3r \
  -H "Authorization: Bearer pf_..."
```

```json
{
  "id": "job_01hx4m2p7n9q6t3r",
  "video_id": "vid_01hx5n3q8m7r2k1p",
  "user_id": "usr_01hx1a2b3c4d5e6f",
  "status": "processing",
  "progress_pct": 47,
  "error": null,
  "created_at": "2024-06-01T12:00:00Z",
  "updated_at": "2024-06-01T12:01:15Z"
}
```

## Job status lifecycle

A job moves through the following states from upload to completion:

```text
queued → processing → done
                    ↘ failed
```

| Status | Description |
| --- | --- |
| `queued` | The video has been received and is waiting for an available worker |
| `processing` | Indexing is actively in progress; check `progress_pct` (0–100) for granular progress |
| `done` | Indexing is complete; the video is fully searchable |
| `failed` | Processing encountered an unrecoverable error; check the `error` field for details |

### Job fields

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | Unique identifier for the job |
| `video_id` | string | ID of the video being processed |
| `user_id` | string | ID of the account that submitted the job |
| `status` | string | Current job state: `queued`, `processing`, `done`, or `failed` |
| `progress_pct` | integer (0–100) | Processing progress percentage |
| `error` | string \| null | Error message if the job `failed`; otherwise `null` |
| `created_at` | string (ISO 8601) | Timestamp when the job was created |
| `updated_at` | string (ISO 8601) | Timestamp of the most recent status update |

<Tip>
  Use the `webhook_url` parameter when uploading to receive a POST notification the moment processing finishes, instead of repeatedly polling the jobs endpoint.

  ```bash
  curl -X POST https://api.pureframe.ai/v1/videos \
    -H "Authorization: Bearer pf_..." \
    -F "file=@/path/to/video.mp4" \
    -F "collection_id=col_01hx3k9z2f8r4e7w" \
    -F "webhook_url=https://yourapp.example.com/webhooks/pureframe"
  ```
</Tip>

## Video metadata

Once a job reaches `done` status, the video object includes the following metadata fields:

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | Unique identifier for the video |
| `filename` | string | Original filename of the uploaded file |
| `collection_id` | string | ID of the collection this video belongs to |
| `status` | string | Processing status of the video |
| `progress_pct` | integer (0–100) | Indexing progress percentage |
| `duration_secs` | number | Length of the video in seconds |
| `width` | integer | Frame width in pixels |
| `height` | integer | Frame height in pixels |
| `fps` | number | Frames per second |
| `codec` | string | Video codec (e.g., `h264`, `vp9`) |
| `mime_type` | string | MIME type of the source file (e.g., `video/mp4`) |
| `file_size_bytes` | integer | Original upload file size in bytes |
| `storage_bytes` | integer | Storage consumed after processing |
| `segments_count` | integer | Number of indexed segments generated from the video |
| `thumbnail_url` | string | URL of an auto-generated thumbnail image |
| `playback_url` | string | Presigned URL for streaming playback (valid approximately 1 hour) |
| `created_at` | string (ISO 8601) | Timestamp when the video was uploaded |

## Search modes enabled after indexing

Indexing a video enables three complementary search modes, each targeting a different layer of content:

<AccordionGroup>
  <Accordion title="video — visual frame-level indexing">
    PureFrame samples frames throughout the video and generates visual embeddings for each. This lets you find moments that _look_ like a query image or match a visual description.
  </Accordion>

  <Accordion title="transcript — speech-to-text indexing">
    PureFrame transcribes all spoken audio and indexes the resulting text. Use this mode to find moments where specific words or phrases are spoken.
  </Accordion>

  <Accordion title="scene — scene-level visual groupings">
    PureFrame detects scene boundaries and creates a higher-level visual summary for each scene. This helps surface broader contextual matches when frame-level precision isn't required.
  </Accordion>
</AccordionGroup>

## Deleting a video

<Warning>
  Deleting a video **permanently removes the video file and all of its indexed segments**. Search results will no longer return any moments from the deleted video. This action cannot be undone.
</Warning>

```bash
curl -X DELETE https://api.pureframe.ai/v1/videos/vid_01hx5n3q8m7r2k1p \
  -H "Authorization: Bearer pf_..."
```
