PureFrame's search finds the exact moment in any video — not just the file. Whether you describe a scene in plain language, paste a reference image URL, or combine both, PureFrame scans visual frames, spoken words, and scene boundaries simultaneously to surface the most relevant moments ranked by semantic similarity.

## Search inputs

Every search request to `POST /v1/search` requires at least one input. You can combine a text query and an image reference in a single request for more precise results.

| Parameter | Type | Description |
|---|---|---|
| `query` | string | Natural language description or keyword string |
| `image` | file | An uploaded image file used as a visual reference |
| `image_url` | string | URL of a reference image for visual similarity matching |

**Text-only search:**

```bash
curl -X POST https://api.pureframe.ai/v1/search \
  -H "Authorization: Bearer pf_..." \
  -H "Content-Type: application/json" \
  -d '{
    "query": "someone opening a cardboard box",
    "natural_language": true
  }'
```

**Combined text + image search:**

```bash
curl -X POST https://api.pureframe.ai/v1/search \
  -H "Authorization: Bearer pf_..." \
  -H "Content-Type: application/json" \
  -d '{
    "query": "product close-up on white background",
    "image_url": "https://example.com/reference-shot.jpg"
  }'
```

## Search modes

Use the `modes` parameter to control which indexes PureFrame queries. It accepts an array of mode names and defaults to all three when omitted.

```json
"modes": ["video", "transcript", "scene"]
```

| Mode | What it searches | Best for |
|---|---|---|
| `video` | Visual embeddings at the frame level | Finding visually similar moments, specific objects, actions, or scenes |
| `transcript` | Speech-to-text index | Finding moments where specific words or phrases are spoken |
| `scene` | Scene-level visual groupings | Broader contextual matches across longer narrative segments |

**Search only transcript and scene indexes:**

```bash
curl -X POST https://api.pureframe.ai/v1/search \
  -H "Authorization: Bearer pf_..." \
  -H "Content-Type: application/json" \
  -d '{
    "query": "quarterly earnings discussion",
    "modes": ["transcript", "scene"]
  }'
```

## Ranking strategies

The `search_mode` parameter controls how PureFrame ranks results when multiple indexes are queried.

<AccordionGroup>
  <Accordion title="image_first (default)">
    Prioritizes visual similarity scores and blends in text signals as secondary factors. Use this when visual relevance is the primary goal — for example, finding a specific look, setting, or on-screen object.
  </Accordion>
  <Accordion title="fusion">
    Combines visual and text rankings using **Reciprocal Rank Fusion (RRF)**, giving each signal equal influence over the final order. Use this when your query has both strong visual and textual components and you want a balanced result set.
  </Accordion>
</AccordionGroup>

```json
"search_mode": "fusion"
```

## Natural language flag

Set `natural_language` to `true` to tell PureFrame to interpret your `query` as a natural language description rather than a keyword string. This improves results for conversational or descriptive queries.

```json
"natural_language": true,
"query": "a person laughing at a dinner table"
```

When `natural_language` is `false` (the default), PureFrame treats the query as a keyword or phrase match and weights exact token overlap more heavily.

## Filtering by labels

Pass a `labels` array to restrict results to segments that match specific semantic labels. Labels let you narrow a broad query to a particular content category without changing the query itself.

```json
"labels": ["outdoor", "interview"]
```

## Paginating results

Use `page` and `per_page` to paginate through result sets. `per_page` defaults to `10`.

```json
"page": 2,
"per_page": 25
```

## Result structure

A successful search response returns an array of `SearchResult` objects. Each result represents a matched video, and each video contains a `segments` array listing the specific moments that matched.

```json
[
  {
    "video_id": "vid_01hx5n3q8m7r2k1p",
    "filename": "product-launch-recap.mp4",
    "collection_id": "col_01hx3k9z2f8r4e7w",
    "segments": [
      {
        "timestamp_start": 42.1,
        "timestamp_end": 55.8,
        "score": 0.94,
        "type": "video",
        "text_content": null,
        "video_url": "https://cdn.pureframe.ai/segments/seg_abc123?token=...",
        "thumbnail_url": "https://cdn.pureframe.ai/thumbnails/seg_abc123.jpg"
      },
      {
        "timestamp_start": 108.0,
        "timestamp_end": 121.4,
        "score": 0.87,
        "type": "transcript",
        "text_content": "Let me walk you through the unboxing experience.",
        "video_url": "https://cdn.pureframe.ai/segments/seg_def456?token=...",
        "thumbnail_url": "https://cdn.pureframe.ai/thumbnails/seg_def456.jpg"
      }
    ]
  }
]
```

### Segment fields

| Field | Type | Description |
|---|---|---|
| `timestamp_start` | number | Start of the matched segment, in seconds |
| `timestamp_end` | number | End of the matched segment, in seconds |
| `score` | number (0–1) | Semantic similarity score; higher is more relevant |
| `type` | string | Index that produced this match: `video`, `transcript`, or `scene` |
| `text_content` | string \| null | Transcribed text for `transcript` segments; `null` for visual segments |
| `video_url` | string | Presigned URL to stream or download the segment clip (valid ~1 hour) |
| `thumbnail_url` | string | URL of a thumbnail image for the segment |

<Tip>
  Improve ranking quality over time by submitting relevance feedback. Send a `POST` request to `/v1/search/feedback` with the search ID and a relevance signal for specific results. PureFrame uses this feedback to tune future rankings for your account.

  ```bash
  curl -X POST https://api.pureframe.ai/v1/search/feedback \
    -H "Authorization: Bearer pf_..." \
    -H "Content-Type: application/json" \
    -d '{
      "search_id": "srch_01hx6p4r9n8s5j2m",
      "segment_id": "seg_abc123",
      "relevant": true
    }'
  ```
</Tip>
