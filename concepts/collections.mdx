Collections are the top-level organizational unit in PureFrame. Every video you upload belongs to a collection, and every search can be scoped to one. Use collections to keep your video library structured, isolate data between projects or customers, and make sure searches return only the content you care about.

## Collection fields

PureFrame returns different sets of fields depending on whether you are listing collections or fetching a single collection's details.

**List item** — returned when listing all collections (`GET /v1/collections`):

| Field | Type | Description |
|---|---|---|
| `id` | string | Unique identifier for the collection |
| `user_id` | string | ID of the account that owns the collection |
| `name` | string | Human-readable name (unique per account) |
| `video_count` | integer | Number of videos currently in the collection |
| `thumbnail_urls` | array of strings | Thumbnail images drawn from recent videos in the collection |
| `created_at` | string (ISO 8601) | Timestamp when the collection was created |

**Detail** — returned when fetching a single collection (`GET /v1/collections/{id}`). Includes all list-item fields, plus:

| Field | Type | Description |
|---|---|---|
| `total_duration_secs` | number | Combined duration of all videos, in seconds |
| `storage_bytes` | integer | Total storage used by the collection |

Collection names must be unique within your account and between 1 and 255 characters long.

## When to use multiple collections

You are not limited to a single collection. Consider creating separate collections when you need clear boundaries between data sets. Common patterns include:

- **Per customer** — give each of your customers their own isolated collection so their videos and search results never mix
- **Per project** — keep distinct projects, campaigns, or productions cleanly separated
- **Per environment** — maintain separate collections for `development`, `staging`, and `production`
- **Content categories** — split your library by content type, such as `interviews`, `tutorials`, or `product-demos`

## Creating a collection

Send a `POST` request to `/v1/collections` with a JSON body containing the collection `name`.

```bash
curl -X POST https://api.pureframe.ai/v1/collections \
  -H "Authorization: Bearer pf_..." \
  -H "Content-Type: application/json" \
  -d '{"name": "product-demos"}'
```

A successful response returns the new collection object:

```json
{
  "id": "col_01hx3k9z2f8r4e7w",
  "user_id": "usr_01hx1a2b3c4d5e6f",
  "name": "product-demos",
  "video_count": 0,
  "thumbnail_urls": [],
  "created_at": "2024-06-01T12:00:00Z"
}
```

<Note>
  Collection names must be unique per account and between 1 and 255 characters long. Attempting to create a collection with a name that already exists in your account returns a `409 Conflict` error.
</Note>

## Scoping searches to a collection

Pass `collection_id` in your search request to restrict results to a single collection. When you omit it, PureFrame searches across all collections in your account.

```bash
curl -X POST https://api.pureframe.ai/v1/search \
  -H "Authorization: Bearer pf_..." \
  -F "query=product unboxing" \
  -F "collection_id=col_01hx3k9z2f8r4e7w"
```

## Retrieving collection details

Fetch the current state of a collection — including its video count, total duration, and storage consumption — with a `GET` request:

```bash
curl https://api.pureframe.ai/v1/collections/col_01hx3k9z2f8r4e7w \
  -H "Authorization: Bearer pf_..."
```

```json
{
  "id": "col_01hx3k9z2f8r4e7w",
  "user_id": "usr_01hx1a2b3c4d5e6f",
  "name": "product-demos",
  "video_count": 42,
  "thumbnail_urls": [
    "https://cdn.pureframe.ai/thumbnails/vid_abc.jpg",
    "https://cdn.pureframe.ai/thumbnails/vid_def.jpg"
  ],
  "total_duration_secs": 18340,
  "storage_bytes": 10737418240,
  "created_at": "2024-06-01T12:00:00Z"
}
```

## Deleting a collection

<Warning>
  Deleting a collection **permanently removes all videos and indexed segments** it contains. This action cannot be undone and will immediately free the associated storage.
</Warning>

Send a `DELETE` request to `/v1/collections/{id}` to remove a collection and all of its contents:

```bash
curl -X DELETE https://api.pureframe.ai/v1/collections/col_01hx3k9z2f8r4e7w \
  -H "Authorization: Bearer pf_..."
```
