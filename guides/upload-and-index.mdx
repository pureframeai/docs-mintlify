PureFrame processes your videos by breaking them into searchable segments — visual frames, scenes, and transcript snippets — and indexing them for semantic search. This guide walks you through uploading a video file, monitoring its indexing job, and confirming it's ready to query.

## Prerequisites

Before you upload your first video, make sure you have:

- A PureFrame API key (starts with `pf_`). Find yours in the [PureFrame dashboard](https://app.pureframe.ai/settings/api-keys).
- At least one collection created. If you haven't created one yet, follow Step 1 below.

---

## Steps

<Steps>
  <Step title="Choose or create a collection">
    Videos live inside collections, which act as isolated search namespaces. If you already have a collection, grab its `collection_id` from the dashboard or from `GET /v1/collections` and skip to Step 2.

    To create a new collection, send a `POST /v1/collections` request:

    ```bash
    curl -X POST https://api.pureframe.ai/v1/collections \
      -H "Authorization: Bearer pf_your_key" \
      -H "Content-Type: application/json" \
      -d '{"name": "My First Collection"}'
    ```

    The response returns a `collection_id` you'll use in the next step:

    ```json
    {
      "data": {
        "collection_id": "col_abc123",
        "name": "My First Collection",
        "created_at": "2024-11-01T10:00:00Z"
      }
    }
    ```
  </Step>

  <Step title="Upload the video">
    Send the video file as `multipart/form-data` to `POST /v1/upload`. Include your `collection_id` and the `file` field pointing to your local file.

    ```bash
    curl -X POST https://api.pureframe.ai/v1/upload \
      -H "Authorization: Bearer pf_your_key" \
      -F "collection_id=col_abc123" \
      -F "file=@/path/to/video.mp4"
    ```

    You can optionally include a `webhook_url` to receive a notification when processing completes instead of polling:

    ```bash
    curl -X POST https://api.pureframe.ai/v1/upload \
      -H "Authorization: Bearer pf_your_key" \
      -F "collection_id=col_abc123" \
      -F "file=@/path/to/video.mp4" \
      -F "webhook_url=https://yourapp.com/webhooks/pureframe"
    ```

    A successful upload returns a `job_id` and `video_id` immediately — processing happens asynchronously:

    ```json
    {
      "data": {
        "job_id": "job_def456",
        "video_id": "vid_ghi789",
        "status": "queued"
      },
      "meta": {
        "request_id": "req_xyz000",
        "timestamp": "2024-11-01T10:01:00Z"
      }
    }
    ```

    Save both `job_id` and `video_id` — you'll use `job_id` to track progress and `video_id` to query the video once it's indexed.
  </Step>

  <Step title="Track indexing progress">
    Poll `GET /v1/jobs/{job_id}` to check the current processing status. The `progress_pct` field gives you a 0–100 completion percentage.

    ```python
    import time, requests

    job_id = "job_def456"
    headers = {"Authorization": "Bearer pf_your_key"}

    while True:
        r = requests.get(
            f"https://api.pureframe.ai/v1/jobs/{job_id}",
            headers=headers
        ).json()

        job = r["data"]
        print(f"Status: {job['status']} ({job['progress_pct']}%)")

        if job["status"] in ("done", "failed"):
            break

        time.sleep(3)
    ```

    The `status` field cycles through these values:

    | Status | Meaning |
    |--------|---------|
    | `queued` | Job is waiting to be picked up by a worker |
    | `processing` | Frames, scenes, and transcript are being extracted |
    | `indexing` | Embeddings are being written to the search index |
    | `done` | Video is fully indexed and searchable |
    | `failed` | Processing failed — check the `error` field |

    <Note>
      Use webhooks to avoid polling — see [Webhooks](/guides/webhooks) for how to register an endpoint and receive `video.processed` events automatically.
    </Note>
  </Step>

  <Step title="Confirm the video is ready">
    Once the job status is `done`, call `GET /v1/videos/{video_id}` to verify indexing succeeded and retrieve playback metadata:

    ```bash
    curl https://api.pureframe.ai/v1/videos/vid_ghi789 \
      -H "Authorization: Bearer pf_your_key"
    ```

    A fully indexed video returns:

    ```json
    {
      "data": {
        "video_id": "vid_ghi789",
        "collection_id": "col_abc123",
        "status": "done",
        "segments_count": 142,
        "duration_seconds": 284,
        "playback_url": "https://cdn.pureframe.ai/videos/vid_ghi789/manifest.m3u8",
        "created_at": "2024-11-01T10:01:00Z",
        "indexed_at": "2024-11-01T10:06:22Z"
      }
    }
    ```

    When `status` is `done` and `segments_count` is greater than zero, the video is fully searchable. Use `playback_url` to embed or stream the video in your application.
  </Step>
</Steps>

## Batch uploading

To upload multiple videos, loop through your files and call `POST /v1/upload` for each one. Collect the returned `job_id` values, then poll each job independently:

```python
import time, requests

video_files = ["clip1.mp4", "clip2.mp4", "clip3.mp4"]
headers = {"Authorization": "Bearer pf_your_key"}
collection_id = "col_abc123"

# Upload all files
jobs = {}
for path in video_files:
    with open(path, "rb") as f:
        resp = requests.post(
            "https://api.pureframe.ai/v1/upload",
            headers=headers,
            files={"file": f},
            data={"collection_id": collection_id}
        ).json()
    job_id = resp["data"]["job_id"]
    video_id = resp["data"]["video_id"]
    jobs[job_id] = {"video_id": video_id, "file": path}
    print(f"Queued {path} → job {job_id}")

# Poll until all jobs complete
pending = set(jobs.keys())
while pending:
    for job_id in list(pending):
        r = requests.get(
            f"https://api.pureframe.ai/v1/jobs/{job_id}",
            headers=headers
        ).json()
        status = r["data"]["status"]
        print(f"{jobs[job_id]['file']}: {status}")
        if status in ("done", "failed"):
            pending.remove(job_id)
    if pending:
        time.sleep(5)

print("All jobs finished.")
```

<Tip>
  For large batches, consider using a `webhook_url` per upload so your server is notified as each video finishes, rather than running a polling loop.
</Tip>

## Error handling

If a job's `status` is `failed`, inspect the `error` field on the job object for a machine-readable code and human-readable message:

```json
{
  "data": {
    "job_id": "job_def456",
    "status": "failed",
    "progress_pct": 34,
    "error": {
      "code": "unsupported_codec",
      "message": "The video stream uses a codec that PureFrame cannot process. Re-encode to H.264 and retry."
    }
  }
}
```

Common error codes:

| Code | Description |
|------|-------------|
| `unsupported_codec` | Re-encode the video to H.264/AAC |
| `file_too_large` | File exceeds the 10 GB per-file limit |
| `corrupt_file` | The file could not be demuxed; verify it plays locally |
| `collection_not_found` | The `collection_id` doesn't exist or belongs to another account |

## Supported formats

PureFrame accepts the following video container formats:

<CardGroup cols={3}>
  <Card title="MP4" icon="file-video">
    H.264 or H.265 video stream
  </Card>
  <Card title="MOV" icon="file-video">
    QuickTime container
  </Card>
  <Card title="AVI" icon="file-video">
    Audio Video Interleave
  </Card>
  <Card title="WebM" icon="file-video">
    VP8, VP9, or AV1 stream
  </Card>
  <Card title="MPEG" icon="file-video">
    MPEG-1 and MPEG-2
  </Card>
  <Card title="MKV" icon="file-video">
    Matroska container
  </Card>
</CardGroup>

<Warning>
  Files must be under **10 GB** and shorter than **6 hours**. If your video exceeds these limits, split it into segments before uploading.
</Warning>
