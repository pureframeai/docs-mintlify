PureFrame exposes an agent-friendly API designed for LLM integrations. Any model that supports function calling or tool use — including GPT-4o, Claude, Gemini, and open-source models via frameworks like LangChain or LlamaIndex — can discover and invoke PureFrame capabilities through a standard OpenAI-compatible tool schema. This means you can give your agent the ability to search videos, inspect collections, and retrieve metadata without writing custom adapter code.

## Available agent tools

PureFrame exposes the following tools through the agent API:

<CardGroup cols={2}>
  <Card title="search_videos" icon="magnifying-glass">
    Run a semantic search across a collection using a text query, an image URL, or both. Returns ranked segments with thumbnail images.
  </Card>

  <Card title="list_collections" icon="layer-group">
    List all collections in the account, with names, video counts, and creation dates.
  </Card>

  <Card title="get_collection" icon="folder-open">
    Retrieve metadata and statistics for a specific collection by ID.
  </Card>

  <Card title="get_video" icon="film">
    Fetch metadata for a single video, including status, duration, segment count, and playback URL.
  </Card>
</CardGroup>

## How it works

The agent integration uses two endpoints:

<Steps>
  <Step title="Fetch the tool schema">
    `GET /v1/agent/schema.json` returns a JSON array of tool definitions in the OpenAI function-calling format. **This endpoint does not require authentication**, so you can fetch and cache the schema at startup without exposing your API key.

    ```bash
    curl https://api.pureframe.ai/v1/agent/schema.json
    ```

    The response is a JSON array ready to pass directly to `tools` in a chat completion call:

    ```json
    [
      {
        "type": "function",
        "function": {
          "name": "search_videos",
          "description": "Search for video segments matching a text query or reference image.",
          "parameters": {
            "type": "object",
            "properties": {
              "query": { "type": "string", "description": "Natural language search query" },
              "collection_id": { "type": "string", "description": "ID of the collection to search" },
              "image_url": { "type": "string", "description": "Optional URL of a reference image" }
            },
            "required": ["collection_id"]
          }
        }
      }
    ]
    ```
  </Step>
  <Step title="Invoke a tool">
    When the model decides to call a PureFrame tool, forward the call to `POST /v1/agent/call` with the tool name and the model's parsed arguments. This endpoint **requires a Bearer API key**.

    ```bash
    curl -X POST https://api.pureframe.ai/v1/agent/call \
      -H "Authorization: Bearer pf_your_key" \
      -H "Content-Type: application/json" \
      -d '{
        "tool": "search_videos",
        "input": {
          "query": "someone waving goodbye",
          "collection_id": "col_abc123"
        }
      }'
    ```

    PureFrame executes the tool and returns a structured result you can pass back to the model as a tool message.
  </Step>
</Steps>

<Note>
  `GET /v1/agent/schema.json` does not require authentication — fetch it freely at startup or build time. `POST /v1/agent/call` requires a `Bearer` API key in the `Authorization` header.
</Note>

## OpenAI function calling integration

The example below wires PureFrame tools into a GPT-4o conversation. The model decides when to call a tool, and you relay the call to PureFrame and inject the result back into the conversation.

```python
import json
import openai
import requests

# 1. Fetch PureFrame tool definitions (no auth needed)
schema_resp = requests.get("https://api.pureframe.ai/v1/agent/schema.json")
schema_resp.raise_for_status()
pf_tools = schema_resp.json()

client = openai.OpenAI()

messages = [
    {"role": "user", "content": "Find clips of someone waving in my videos"}
]

# 2. Send the initial request with PureFrame tools attached
chat_resp = client.chat.completions.create(
    model="gpt-4o",
    messages=messages,
    tools=pf_tools
)

assistant_message = chat_resp.choices[0].message

# 3. If the model called a tool, forward it to PureFrame
if assistant_message.tool_calls:
    tool_call = assistant_message.tool_calls[0]

    pf_result = requests.post(
        "https://api.pureframe.ai/v1/agent/call",
        headers={"Authorization": "Bearer pf_your_key"},
        json={
            "tool": tool_call.function.name,
            "input": json.loads(tool_call.function.arguments)
        }
    )
    pf_result.raise_for_status()
    tool_output = pf_result.json()

    # 4. Append the assistant turn and tool result, then get a final reply
    messages.append(assistant_message)
    messages.append({
        "role": "tool",
        "tool_call_id": tool_call.id,
        "content": json.dumps(tool_output)
    })

    final_resp = client.chat.completions.create(
        model="gpt-4o",
        messages=messages,
        tools=pf_tools
    )
    print(final_resp.choices[0].message.content)
```

## Using thumbnail images with vision models

`search_videos` results include a `thumbnail_base64` field — a base64-encoded JPEG of the matching frame. Vision-capable models like GPT-4o and Claude can consume this image directly, letting the agent visually reason about the results without making an additional network request.

```python
import json, base64
import openai, requests

# Assume tool_output is the parsed response from POST /v1/agent/call
segments = tool_output["data"]["results"][0]["segments"]
top_segment = segments[0]

thumbnail_b64 = top_segment["thumbnail_base64"]  # Already a base64 JPEG string
timestamp = top_segment["timestamp_start"]
video_id = tool_output["data"]["results"][0]["video_id"]

# Pass the thumbnail directly to GPT-4o as a vision message
client = openai.OpenAI()

vision_resp = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "text",
                    "text": f"This is a frame from video {video_id} at {timestamp}s. Describe what is happening."
                },
                {
                    "type": "image_url",
                    "image_url": {
                        "url": f"data:image/jpeg;base64,{thumbnail_b64}"
                    }
                }
            ]
        }
    ]
)

print(vision_resp.choices[0].message.content)
```

<Tip>
  If you're building a multi-step agent that re-ranks or filters results before showing them to the user, pass `thumbnail_base64` images to the model at the filtering step so it can use visual reasoning to pick the best match.
</Tip>

## Using PureFrame with Claude

The PureFrame tool schema is compatible with Anthropic's tool-use format with a small adaptation. Claude expects tools wrapped in an `input_schema` key instead of `parameters`:

```python
import anthropic, requests, json

schema_resp = requests.get("https://api.pureframe.ai/v1/agent/schema.json").json()

# Convert OpenAI format → Anthropic format
claude_tools = [
    {
        "name": t["function"]["name"],
        "description": t["function"]["description"],
        "input_schema": t["function"]["parameters"]
    }
    for t in schema_resp
]

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-opus-4-5",
    max_tokens=1024,
    tools=claude_tools,
    messages=[{"role": "user", "content": "Find clips of someone waving in my videos"}]
)

# Forward any tool calls to PureFrame the same way as the OpenAI example
```
